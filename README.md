## 🛠️ 原因分析与技术总结

### 1. 路径与命名的"精准打击"

- **问题**：跳转失败多因文件名使用了点号（如 `2025.12.28.1`）、缺失 `.html` 后缀或代码中存在多余空格。

- **经验**：全栈开发中，文件名的大小写、后缀名以及路径字符串的**绝对匹配**是资源加载的首要前提。

### 2. 邮件报警链路的"排坑指南"

- **格式陷阱**：直接使用 `echo` 发送邮件易导致换行符解析失效。

- **对策**：改用 `printf` 严格构造包含 `To:`、`Subject:` 和空行的标准 SMTP 数据包。

- **退信抑制 (Suppression List)**：由于初期未配置 Cloudflare **Catch-all** 导致投递失败，IP 被 Resend 临时拉黑。

- **对策**：开启万能转发规则并手动从 Resend 抑制列表中移除地址，最终打通链路。

### 3. 前端性能与品牌一致性

- **图标规范**：大尺寸图片（1024px）作为 Favicon 会拖慢 FCP（首次内容绘制）。优化为 64px 后，资源请求更轻量。

- **资源架构**：通过 `https://yushenchuanmei.top/favicon.ico` 建立单一事实来源，降低了多仓库间冗余资产的维护成本。

### 4. Git 与版本控制最佳实践

- **代理问题**：异常代理端口会导致 `Connection Reset`，需及时清除 `http.proxy` 和 `https.proxy` 配置。

- **SSH vs HTTPS**：SSH 密钥认证更安全高效，推荐作为主要推送方式，HTTPS 作为备用。

- **双远程库策略**：同时配置 SSH 和 HTTPS 远程地址，确保任何网络环境下都能推送。

## 💡 核心感悟

> **运维是地基**：即便是一个简单的静态页，也需要可靠的监控（如 SSH 报警）来确保环境安全。

> **细节决定成败**：一个多余的 `.png` 后缀或一个缺失的换行符，都可能导致整个系统报错。

> **审美也是战斗力**：Three.js 的加入让单纯的日志变成了具有交互体验的作品。

> **工程化思维**：规范的目录结构和自动化脚本是高效开发的基石。
